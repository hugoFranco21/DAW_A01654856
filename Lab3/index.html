Definiciones

URL: Son las siglas en inglés de “Uniform Resource Locator”, es una dirección única que poseen todos los elementos en Internet, es decir que literalmente existe una URL por cada página, imagen, video y documento en Internet, lo que nos permitirá ubicarlos fácilmente gracias que la información que contiene. [1]

MIME: El tipo Extensiones multipropósito de Correo de Internet (MIME) es una forma estandarizada de indicar la naturaleza y el formato de un documento. Los navegadores a menudo usan el tipo MIME para determinar cómo procesará un documento; por lo tanto, es importante que los servidores estén configurados correctamente para adjuntar el tipo MIME correcto al encabezado del objeto de respuesta. [2]

Metodos HTTP: [3]

  • GET: Significa recuperar cualquier información (en forma de una entidad) identificada por el Request-URI. Cabe señalar que todas las peticiones que usan el método GET, deberán recuperar únicamente datos.
  • OPTIONS: Representa una solicitud de información acerca de las opciones de comunicación disponibles en el canal de solicitud/respuesta identificada por el Request-URI.
  • HEAD: Es muy similar al GET, a excepción de que el servidor responde con líneas y headers, pero no con el body de la respuesta.
  • PUT: Es usado para solicitar que el servidor almacene el cuerpo de la entidad en una ubicación específica dada por el URL.
  • POST: Es usado cuando se requiere enviar información al servidor como, por ejemplo, un archivo de actualización, información de formulario, etc.
  • DELETE: Es utilizado para solicitar al servidor que elimine un archivo en una ubicación específica dada por la URL.
  • CONNECT: Es usado por el cliente para establecer una conexión de red con un servidor web mediante HTTP misma que se establece en forma de un túnel.
  • TRACE: Se utiliza para realizar pruebas de eco (de retornos) de mensajes en el camino que existe hacia un recurso determinado.

XML: Sus siglas vienen de Extensible Markup Language, es un formato de texto muy flexible que fue diseñado para safrontar los desafios de la publicacion electronica a gran escala. [5]

XHTML: Sus siglas significan Extensible HyperText Markup Language, es practicamente identico a HTML, la mayor diferencia siendo que este es mucho mas estricto en cuanto a la sintaxis. Todos los navegadores soporta XHTML. [6]

HTML5: Es la version mas reciente del lenguaje utilizado para el despliegue de paginas web. Utiliza nuevas etiquetas cuyo enfoque es el de añadir contenido multimedia. [7]

JavaScript: Es un lenguaje de programacion orientado a objetos, interpretado, conocido por ser el lenguaje de scripting de las paginas web, aunque tambien se utiliza en entornos no web. Corre del lado del cliente, lo que se utiliza para diseñar el comportamiento de las paginas web ante la ocurrencia de un evento. [8]

PHP: Acronimo de HyperText Preprocessor, es un lenguaje de codigo abierto utilizado para el desarrollo web y que puede ser incrustado en HTML. Lo que distingue a PHP de algo del lado del cliente como Javascript es que el código es ejecutado en el servidor, generando HTML y enviándolo al cliente. [9]

JSP: Acronimo de JavaServer Pages, sirve para crear y gestionar paginas web dinamicas, usando HTML para generar la parte estatica y Java para la parte dinamica. El objetivo final es separar la interfaz de la implementacion. [10]

AJAX: Acronimo de Asynchronous JavaScript and XML, no es un lenguaje de programacion, es una combinacion de objetos XMLHttpRequest y JavaScript, que permite a una pagina web ser actualizada asincronamente al interactuar con el servidor sin tener que recargar la pagina. [11]


Preguntas Lab. 1

  1. ¿Cual es la diferencia entre Internet y la World Wide Web?
    La World Wide Web es un modelo para compartir informacion que esta construido sobre Internet, utiliza el protocolo HTTP y es como se accede a los sitios web. Internet es una red global a gran escala que permite la conexion de millones de dispositivos al mismo tiempo, y usa diversos protocolos para la transferencia de informacion. [12]
  2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? [3]
    • GET: Significa recuperar cualquier información (en forma de una entidad) identificada por el Request-URI. Cabe señalar que todas las peticiones que usan el método GET, deberán recuperar únicamente datos.
    • HEAD: Es muy similar al GET, a excepción de que el servidor responde con líneas y headers, pero no con el body de la respuesta.
    • POST: Es usado cuando se requiere enviar información al servidor como, por ejemplo, un archivo de actualización, información de formulario, etc.
    • PUT: Es usado para solicitar que el servidor almacene el cuerpo de la entidad en una ubicación específica dada por el URL.
    • PATCH: El método HTTP PATCH se utiliza para realizar una actualización parcial de un recurso de OSLC. A diferencia del método PUT, el método PATCH no suprime las propiedades de recurso local que no están incluidas en la petición.[13]
    • DELETE: Es utilizado para solicitar al servidor que elimine un archivo en una ubicación específica dada por la URL.

  5. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    Los codigos 2XX indican que la peticion ha sido correctamente recibida y procesada, especificamente el codigo 200 es OK, y significa que el request fue correcto, es la respuesta estandar. [4]
  6. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    El error 404 indica que el recurso no pudo ser encontrado, y esta clasificado como un error de usuario. Lo mas probable es que el usuario haya escrito incorrectamente la URL, o que la pagina a la que desea entrar aun no este disponible, por lo que no seria responsabilidad del desarollador.
  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    El error 500 indica una falla generica del servidor por la que no haya sido capaz de procesar la peticion. Si la falla se debe a como esta configurado el servidor entonces si es responsabilidad del desarrollador, si hubo algun otro problema (falla electrica, etc.) no se puede responsabilizar al desarollador.
  8. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    Son atributos a los cuales en las nuevas versiones de HTML5 ya no se le da soporte y ha sido reemplazado por alguno de otro nombre o eliminado en su totalidad.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?
El ciclo de vida comprende las siguientes etapas: [19]

  • Planificación
  • Analisis
  • Diseño
  • Implementacion
  • Pruebas
  • Instalacion o despliegue
  • Uso y mantenimiento

¿Cuál es el ciclo de desarrollo de sistemas de información? El ciclo de desarrollo comprende las siguientes etapas: [20]

  • Planificacion del concepto
  • Definicion de requisitos
  • Diseño
  • Fase del desarrollo y pruebas
  • Puesta en marcha
  • Operaciones y mantenimientos
  • Disposicion
NOTA: La mayoria de la informacion que encontre sobre el ciclo de vida y ciclo de desarrollo era practicamente identica.

Preguntas Lab. 3

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    Creo que el !important es un comando muy poderoso, el cual solo debe ser usado en situaciones extraordinarias, y que es mejor evitarlo. Se puede evitar si se realiza un correcto analisis de los estilos a utilizar y saber que tan especifico se quiere definir el estilo de los elementos. [21]
  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    Porque las imagenes suelen tener elementos que dificultan la lectura de texto por el minimo contraste que existe entre los colores. A menos que la imagen haya sido creada con el proposito de actuar como fondo de pagina, es muy dificil que se ajuste el contenido correctamente.
  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    Mi recomendacion es ser consistente. Muchas veces se debe hacer prueba y error en lo que se encuentra un tamaño ideal, pero si ya se esta trabajando con algo es necesario apegarse a ello para que la pagina se vea cohesiva.
  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    Porque al hacerlo se eliminan datos innecesarios o redundantes sin que se afecte la forma en que son procesados por los navegadores, como por ejemplo comentarios, formato, etc. [22]
